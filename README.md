# Модуль системы наведения для CubeSat 1U

Данный репозиторий содержит материалы командного кейса №1 «Модуль системы наведения» в рамках Московской предпрофессиональной олимпиады школьников (профиль «Инженерия», 2025/2026 уч. г.). Разработан программно-аппаратный комплекс – независимый модуль наведения лазерного целеуказателя для космического аппарата форм-фактора CubeSat 1U.

## Цель и задачи

### Цель работы

Разработать автономный модуль наведения лазерного луча в двух осях (наклон/поворот) с углом обзора ±40° по каждой оси, управляемый по радиоканалу, и интегрировать его в корпус CubeSat 1U.

### Задачи

- **Механическая конструкция**: Спроектировать механическую конструкцию, обеспечивающую крепление двух сервоприводов, лазера, контроллера и радиомодуля внутри корпуса 100×100×100 мм.

- **Электрическая схема**: Разработать электрическую схему и собрать макет.

- **Программное обеспечение**: Написать программное обеспечение для RP2040-Zero (спутник) и Arduino Nano (наземная станция), реализующее алгоритмы сканирования (горизонтальное, вертикальное, диагональные) с шагом 10° и задержкой 3 секунды.

- **Радиокоммуникация**: Обеспечить приём команд и передачу телеметрии по радиоканалу (NRF24L01).

## Описание функций

### Основные характеристики системы

Модуль системы наведения позволяет:

- Управлять направлением лазерного луча в двух осях независимо
- Обеспечивать полный охват пространства на ±40° по каждой оси
- Реализовывать несколько режимов сканирования с точностью 10°
- Прин��мать команды по радиоканалу и передавать данные о текущем положении

### Требования к функциональности

**Аппаратная часть включает:**
- Arduino Nano — наземная станция, принимающая команды START от ПК и передающая последовательности позиций по радио
- Raspberry Pi Pico (RP2040) — исполнительный модуль внутри CubeSat, управляющий двумя сервоприводами 9G Digital servo и лазером KY-008
- Радиомодуль nRF24L01 для односторонней связи от Arduino к RP2040
- Собственный 3D-напечатанный корпус, соответствующий стандарту CubeSat 1U

## Алгоритм работы ПО

### Алгоритмы сканирования

1. **Горизонтальное сканирование** (ось Z)
   - Направление: от -40° до +40° с шагом 10°
   - Задержка между позициями: 3 секунды

2. **Вертикальное сканирование** (ось Y)
   - Направление: от -40° до +40° с шагом 10°
   - Задержка между позициями: 3 секунды

3. **Диагональное сканирование**
   - От точки (-40°, -40°) до (40°, 40°) с шагом 10°
   - От точки (-40°, 40°) до (40°, -40°) с шагом 10°

### Управление

Система получает команды от наземной станции и выполняет соответствующие движения с последующей передачей телеметрии.
